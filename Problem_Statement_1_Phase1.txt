Background Context
We have spent the last decade perfecting the vertical scale of isolated models.
We know
 how to build massive context windows, and we know how to orchestrate fragile, brittle
 multi-agent pipelines using hardcoded text routers.
But this is a dead end for true Artificial
 Superintelligence (ASI).
True ASI requires horizontal scale-out - a self-evolving, internet-scale ecosystem where
 heterogeneous Al entities and human specialists don't just pass text tokens back and forth,
 but actually think together.
To achieve this, we need to move past application-layer hacks and build the foundational
 infrastructure: The Internet of Cognition (IOC).
Task
You are tasked with building a resilient, secure mini-prototype of this cognitive network
 layer.
There are no fixed test cases to pass; your success will be measured by the
 architectural trade-offs you make, the technical credibility of your design, and your defense
 of your engineering choices.
Phase 1 submissions will be evaluated by an automated Al engine using organizer-defined
 Phase 1 evaluation criteria.
Teams that clear the qualification threshold should unlock
 Phase 2 immediately.
Teams that do not clear the threshold may still be moved to Phase 2
 later under organizer-controlled progression rules.
Phase 1 is intended to be a compact qualification round. Teams are not expected to build a
 production-ready system here.
A tightly scoped but technically credible handshake
 prototype, simulation, or proof-of-concept is sufficient if it clearly demonstrates the core
 protocol behavior.
The Scenario
Phase 1: The Semantic Handshake & Intent Alignment
When two highly specialized, autonomous Al agents, operating on entirely different
 underlying model architectures and optimization goals, try to collaborate over a standard
 network, they suffer from semantic isolation.
If Agent A is optimizing for ultra-low-latency
 network routing and Agent B is optimizing for maximum zero-trust security compliance,
 they don't just speak different data schemas;
they have conflicting core intents. Passing raw
 text strings or JSON payloads back and forth leads to optimization deadlocks, hallucinated
 boundaries, or catastrophic failures.
Your Task
Design and implement a lightweight, programmatic Cognition State Protocol (CSP) layer.
You must construct a mechanism where two simulation agents with opposing optimization
 targets (for example, Agent 1: Maximize Throughput vs. Agent 2: Maximum Security
 Inspection) can dynamically discover each other's operational states, negotiate their
 constraints, and achieve a stable shared Intent Handshake without a central human-in-the-
 loop coordinator.
The system should establish a unified, machine-readable intent state that both agents agree
 to bound their behavior by.
